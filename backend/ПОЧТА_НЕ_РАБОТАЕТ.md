# Почта не работает — что проверить

## 1. mail.env заполнен реальными данными?

Открой `backend/mail.env`. Если там:
- `SMTP_HOST=smtp.example.com` — **заглушка**, замени на `smtp.gmail.com` / `smtp.mail.ru` / `smtp.yandex.ru`
- `SMTP_USER=your-email@example.com` — **заглушка**, впиши свой реальный email
- `SMTP_PASSWORD=ВСТАВЬ_ПАРОЛЬ_СЮДА` — **заглушка**, впиши пароль или пароль приложения

После правок **перезапусти бэкенд** (Ctrl+C и снова `python run.py`).

---

## 2. Gmail — нужен пароль приложения

Обычный пароль от Gmail не подойдёт, если включена 2FA. Нужен **пароль приложения**:

1. [myaccount.google.com](https://myaccount.google.com) → Безопасность
2. Двухэтапная аутентификация должна быть включена
3. Пароли приложений → Создать → выбрать «Почта» → скопировать 16-символьный пароль
4. В `mail.env` в `SMTP_PASSWORD=` вставить этот пароль (без пробелов)

---

## 3. Mail.ru / Yandex

- **Mail.ru:** Настройки → Пароль и безопасность → Пароль для внешнего приложения
- **Яндекс:** [id.yandex.ru](https://id.yandex.ru) → Пароли приложений

В `mail.env`:
- Mail.ru: `SMTP_HOST=smtp.mail.ru`, `SMTP_PORT=587`
- Yandex: `SMTP_HOST=smtp.yandex.ru`, `SMTP_PORT=587`

---

## 4. Порт 465 vs 587

- **587** — STARTTLS (обычно работает везде)
- **465** — SSL с самого начала (код теперь поддерживает оба)

Если 587 не работает (firewall, провайдер режет), попробуй 465.

---

## 5. Смотри логи бэкенда

При отправке письма при ошибке в логах будет:
```
Send verification email failed: ...
```

Типичные сообщения:
- `Authentication failed` — неверный логин или пароль (для Gmail — используй пароль приложения)
- `Connection refused` / `timed out` — firewall блокирует порт 587/465 или неверный SMTP_HOST
- `SSL: CERTIFICATE_VERIFY_FAILED` — проблема с сертификатами (редко на Linux)

---

## 6. Быстрая проверка

Зарегистрируй тестового пользователя с **реальным** email. Письмо должно прийти в течение минуты. Проверь папку «Спам».
